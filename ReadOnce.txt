1. To find origin link of frontend (for CORS) : open frontend after Nginx setup and in console type : window.location.origin
2. Frontend .env -> localhost (from VITE_API_URL) is replaced by public ip of backend_VM (keep empty if using reverse proxy)
3. Backend .env -> CLIENT_URL = <whatever you get in step 1 of this file (window.location.origin)>

4. Backend .env -> PORT value is the one on which my backend runs so set inbound rule for that in Azure :
| Setting          | Value |
| ---------------- | ----- |
| Source           | Any   |
| Source Port      | *     |
| Destination Port | 5000  |
| Protocol         | TCP   |
| Action           | Allow |

5. Additionally for reverse proxy: sudo nano /etc/nginx/sites-available/default

server {
    listen 80;
    server_name _;

    # Frontend
    location / {
        root /var/www/html;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # Reverse proxy to backend VM
    location /api/ {
        proxy_pass http://<VM2_PUBLIC_IP>:5000;

        proxy_http_version 1.1;

        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
